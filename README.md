# iMage

## Установка на телефон (PWA)

После публикации откройте на телефоне:

`https://image-5e192.web.app/`

- Android/Chrome: нажмите **Установить** в приложении или выберите
  **Установить приложение** в меню браузера.
- iPhone/Safari: нажмите **Поделиться** → **На экран «Домой»**.

Шифрование и обработка изображений выполняются только на устройстве. Python-сервер
PWA не требуется. Аккаунты, роли и настройки хранятся в Firebase Authentication
и Firestore, поэтому доступны на разных устройствах. Шифрование совместимо с
бинарным форматом V2 desktop-версии:
PBKDF2-SHA256 (600 000 итераций) + Fernet.

Секретное изображение скачивается в ZIP, чтобы мессенджер не пересохранил и не
повредил JPG. Во вкладках открытия и поиска можно выбрать сам JPG или ZIP —
`iMage` автоматически распакует первый JPG/PNG из архива прямо в браузере.

Обычный пользователь видит две вкладки: создание секретного JPG и открытие с
редактированием текста. Администратор дополнительно видит поиск пароля и админку.
В шестерёнке доступны светлая/тёмная тема, русский/английский/датский язык,
установка приложения, принудительное обновление и выход.

Программа для Windows, которая шифрует JSON с помощью пароля и прячет его в
обычном JPG-файле. Получившееся изображение продолжает открываться как обычная
фотография. Есть два интерфейса: локальная веб-страница для браузера и
классическое desktop-окно Tkinter. PWA использует Firebase, а desktop-версия
продолжает использовать собственную локальную авторизацию.

## Возможности

- выбор исходного JPG или PNG;
- отсутствие искусственного лимита размера файла (фактический предел зависит от памяти устройства);
- проверка JSON перед созданием секретного изображения;
- получение ключа из пароля через PBKDF2HMAC + SHA-256;
- шифрование JSON с помощью Fernet;
- извлечение и сохранение расшифрованного JSON;
- редактирование извлечённого JSON и создание обновлённого секретного JPG;
- поиск пароля по выбранным наборам вариантов, уровнем скорости 1–10, паузой и отображением прогресса;
- понятные сообщения при неверном пароле или отсутствии секрета.

Пароль изображения нигде не хранится. Пароли аккаунтов обрабатывает Firebase
Authentication: приложение не получает и не показывает старые пароли.

## Firebase в PWA

- вход выполняется по email или логину;
- регистрация создаёт Firebase Auth-аккаунт и документы `users/{uid}` и
  `usernames/{usernameLower}` в Firestore;
- роль и настройки темы/языка читаются из `users/{uid}`;
- администратор может менять роли, удалять Firestore-профили и отправлять письма
  для сброса пароля;
- изображения, секретный текст, пароль изображения и история операций в Firebase
  не отправляются.

## Требования

- Windows 10/11;
- Python 3.10 или новее.

## Установка

Откройте PowerShell или командную строку в папке проекта и выполните:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
```

Если выполнение `Activate.ps1` запрещено политикой PowerShell, виртуальное
окружение можно не активировать:

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

## Запуск в браузере

```powershell
python web_app.py
```

После запуска откройте [http://127.0.0.1:5055](http://127.0.0.1:5055).
Страница работает только локально: изображения и данные не отправляются в
интернет. В браузерной версии достаточно писать обычный текст — программа сама
упаковывает его в корректный JSON перед шифрованием. Разрешены переносы строк,
кавычки и любые символы. Созданное изображение сразу появляется во второй
вкладке, где текст можно раскрыть, отредактировать и снова сохранить.

## Запуск desktop-версии

```powershell
python main.py
```

## Локальная авторизация desktop-версии

При первом запуске создайте локальную учётную запись администратора: укажите
логин и пароль не короче 6 символов. В дальнейшем сначала вводите эти данные на
экране входа. Настройки входа находятся в
`%APPDATA%\SecretImageJSON\auth.json`.

Эти данные относятся только к Tkinter/локальному Python-интерфейсу и не связаны
с аккаунтами Firebase в PWA.

## Поиск пароля в PWA

Во вкладке **03 Поиск пароля** выберите секретный JPG, укажите минимальную и
максимальную длину, отметьте нужные режимы и нажмите **Узнать пароль**. Поиск
можно поставить на паузу, продолжить или остановить. Программа не извлекает
пароль напрямую — она проверяет варианты расшифровкой файла. Из-за 600 000
итераций PBKDF2 полный перебор длинных паролей может занять очень много времени.

Пароль администратора защищает вход в программу, а пароль изображения шифрует
конкретный JSON. Это два независимых пароля. Чтобы сбросить забытые данные
администратора, закройте программу, удалите указанный `auth.json` и снова
запустите приложение — появится экран создания учётной записи.

1. После входа откройте вкладку **Create Secret Image**.
2. Нажмите **Choose Image** и выберите JPG или PNG.
3. Введите JSON в поле **JSON content**.
4. При желании нажмите **Validate JSON**.
5. Введите пароль шифрования и нажмите **Create Secret Image**.
6. Выберите имя и место сохранения нового JPG.

## Как извлечь JSON

1. Откройте вкладку **Open / Edit Secret Image**.
2. Нажмите **Choose Secret Image** и выберите созданный JPG.
3. Введите тот же пароль и нажмите **Extract JSON**.
4. Отредактируйте показанный JSON, если это нужно.
5. Нажмите **Save Updated Secret Image**, чтобы создать новый JPG с изменённым
   секретом, или **Save JSON**, чтобы сохранить обычный `.json`-файл.

## Формат данных

В JPEG сохраняется зашифрованная бинарная копия в служебных блоках APP15, а в
конце файла остаётся совместимая резервная копия формата V2. Данные содержат
случайную 16-байтовую соль и сырой бинарный Fernet-токен. Читаемых меток
`SECRET`, `JSON` или `START` нет. Ключ не хранится в изображении: он заново
получается из пароля и соли с помощью PBKDF2HMAC-SHA256 (600 000 итераций).
Старые файлы формата V1/V2 по-прежнему можно открыть.

Важно: повторное сжатие фотографии может удалить любой встроенный секрет.
Поэтому `iMage` скачивает результат в ZIP. Передавайте ZIP без распаковки;
получатель может выбрать архив прямо во вкладке открытия или поиска.
